Episode 01 / Network Anatomy

The Internet Is Just Packets With Better PR

A computer network is a group of connected devices that exchange data and share resources. Devices are nodes, the paths are links, and protocols are the rules that keep everyone from screaming in different formats.

Node Any device that sends, receives, or forwards data: laptop, mobile, printer, server, router.
Packet A small chunk of data with addressing information. Big files travel as many packets and reassemble later.
Protocol A rulebook for formatting, sending, receiving, acknowledging, and securing data.

Network Devices You Must Know

Router

Connects different networks, uses IP addresses, and chooses the best path for packets.

Switch

Connects devices inside the same network and forwards frames to the intended device instead of shouting to everyone.

Hub

Basic connector that broadcasts data to all ports. Less efficient and less secure than a switch.

Gateway

Translates between networks that use different protocols. Often used at the edge of private networks.

Access Point

Extends a wired network into wireless coverage so mobile devices can join.

Firewall

Monitors and controls traffic based on rules. It allows, blocks, or filters traffic.

Packet Drama

How The Internet Works

Connect

Your device connects through a router or modem to an ISP.

Resolve

DNS translates a domain name like example.com into an IP address.

Handshake

TCP establishes reliability. TLS may secure the session before sensitive data moves.

Request

The browser sends an HTTP/HTTPS request to the server.

Reassemble

HTML, CSS, JavaScript, images, and other data return as packets and are rebuilt by your browser.

OSI Model: The Seven-Layer Cake With Consequences

7 Application

User-facing network services: HTTP/S, DNS, FTP, SMTP.

6 Presentation

Formatting, translation, compression, encryption and decryption.

5 Session

Starts, manages, and terminates communication sessions.

4 Transport

Segmentation, reliability, flow control. TCP and UDP live here.

3 Network

Logical addressing and routing. IP, routers, packets.

2 Data Link

Frames, MAC addresses, node-to-node delivery, local error detection.

1 Physical

Bits moving through cables, fiber, radio, or electrical signals.

TCP handshake overlay
CLIENT -> SERVER : SYN      // can we talk?
SERVER -> CLIENT : SYN-ACK  // yes, I hear you
CLIENT -> SERVER : ACK      // locked in, send data

Exam line: TCP is reliable because it uses sequencing, acknowledgements, checksums, and retransmission.

Episode 02 / Core Security Principles

Trust Is Cute. Verification Is Hot.

Cybersecurity protects networks, devices, identities, data, and systems from unauthorized access, disruption, damage, or misuse. The goal is not magical invincibility. It is reducing exposure, detecting trouble early, and recovering well.

CIA Triad

Confidentiality

Only authorized people or systems can access sensitive information. Think least privilege, MFA, access control, encryption.

Integrity

Data remains accurate, complete, and protected from unauthorized changes. Think hashing, digital signatures, audit logs.

Availability

Systems and data are accessible when needed. Think backups, redundancy, DDoS protection, disaster recovery.

Vought Security Advisory

Do not confuse: Confidentiality hides data, integrity proves data was not messed with, availability keeps data reachable. A ransomware attack often damages availability first, then confidentiality/integrity depending on theft or tampering.

IAM: Identity Access Management

IAM answers two exam-friendly questions: Who are you? and what are you allowed to do? Identity management authenticates the user or device. Access management authorizes resources based on rules, roles, context, and least privilege.

Authentication

Verifies identity using passwords, MFA, biometrics, OTPs, passkeys, or certificates.

Authorization

Grants permissions after identity is verified. RBAC assigns access based on roles.

SSO

Single sign-on lets users access multiple services through one authenticated identity flow.

Zero Trust

Verify explicitly, use least privilege, and assume breach instead of trusting by network location.

Cryptography Without The Wizard Smoke

Symmetric Encryption

Same secret key encrypts and decrypts. Fast and useful for bulk data. Key exchange is the hard part. Example: AES.

Asymmetric Encryption

Public key encrypts or verifies, private key decrypts or signs. Slower but solves key sharing. Example: RSA, ECC.

Hashing

One-way function that turns data into a fixed digest. Used for integrity checks and password storage. Example: SHA-256.

Digital Signature

Proves authenticity and integrity. Sender signs with private key; receiver verifies with public key.

Memory hook
encrypt(message, key) -> ciphertext
hash(message) -> digest
sign(hash, private_key) -> proof of origin + integrity
Episode 03 / Vulnerabilities & Testing

Weakness Is Not The Same As Risk

A vulnerability is a weakness that can be exploited. A threat is something that can exploit it. Risk is the likelihood and impact of that exploitation. This matters because a weakness in a worthless isolated system may be low risk, while a simple flaw in a payment server may be critical.

VulnerabilityA flaw, gap, misconfiguration, bug, weak password, open port, or weak process.
ThreatAn attacker, malware, insider, disaster, or mistake that could cause harm.
RiskProbability multiplied by impact, shaped by asset value and controls.

Common Vulnerability Categories

Software

Input validation flaws, SQL injection, XSS, buffer overflows, unsafe dependencies, missing patches.

Network

Weak authentication, exposed services, insecure architecture, poor segmentation, open ports.

Personnel

Weak passwords, poor awareness, unsafe attachments, careless data handling.

Organizational

No incident response plan, poor audits, weak continuity planning, unclear ownership.

Security Testing Types

Vulnerability Scanning

Automated detection of known weaknesses, missing patches, misconfigurations, and open ports.

Security Scanning

Finds weak points in systems or networks and recommends mitigations.

Penetration Testing

Simulates real attacks to validate whether weaknesses can be exploited.

Risk Assessment

Classifies risks as low, medium, or high and prioritizes controls.

Security Audit

Inspects configurations, code, standards, policies, and control gaps.

Posture Assessment

Combines scanning, ethical hacking, and risk assessment for an overall view.

Pen Test Process

A penetration test is a controlled mock cyberattack. It has permission, scope, rules, reporting, and cleanup. It can be black-box, white-box, or gray-box depending on how much information testers receive.

Reconnaissance

Gather public and technical information about the target.

Discovery

Identify services, ports, vulnerabilities, and possible attack paths.

Exploitation

Safely prove impact by exploiting a weakness within agreed scope.

Escalation

Check whether one foothold can lead to deeper access or privilege abuse.

Cleanup

Remove test artifacts and avoid leaving new weaknesses behind.

Report

Explain findings, evidence, severity, business impact, and remediation steps.

Episode 04 / The Human Layer

The Exploit Wore A Nice Email Subject

Social engineering targets human judgment instead of directly attacking code. It uses trust, urgency, fear, authority, curiosity, or familiarity to make people reveal information or perform unsafe actions.

Most defenses check packets. Social engineering checks whether you are tired.

Social Engineering Flow

Information Gathering

Collect names, roles, emails, public posts, routines, vendors, or procedures.

Trust Building

Pretend to be a known person, support staff, vendor, bank, authority, or friend.

Exploitation

Make the victim click, share OTPs, reset passwords, approve payments, or open files.

Execution

Use the access or information to steal money, data, identity, or system access.

Common Cybercrime Types

Phishing & Email Scams

Fake messages impersonate trusted brands or people to steal credentials or money.

Banking / UPI / Card Fraud

Scams involving payment apps, cards, SIM swap, fake refunds, or transaction deception.

Malware

Software built to disrupt systems, steal data, spy, or create remote control.

Ransomware

Encrypts data and demands payment for restoration, often causing downtime.

Identity Theft

Personal data is stolen and used for fraud, impersonation, or account takeover.

Online Harassment

Cyberstalking, threats, bullying, doxxing, or sharing sensitive content without consent.

India Reporting Note

The National Cyber Crime Reporting Portal supports online cybercrime complaint reporting, especially crimes against women and children. For cyber financial fraud, India uses the helpline 1930. Keep incident date/time, details, identity proof, transaction IDs, suspect details, URLs, and evidence ready where applicable.

Episode 05 / Response & Continuity

Clean Up The Scene. Preserve The Evidence.

DFIR combines digital forensics and incident response. The response side contains the threat. The forensics side preserves and analyzes evidence so the incident can be understood, proven, and prevented next time.

Incident Response Steps

Preparation

Write plans, assign roles, train people, manage vulnerabilities, and prepare tools.

Detection & Analysis

Monitor alerts, triage suspicious activity, validate incidents, and reduce false positives.

Containment

Stop the threat from spreading across the network.

Eradication

Remove malware, attacker access, persistence, and malicious changes.

Recovery

Restore systems, verify clean operation, and return services safely.

Post-Incident Review

Document what happened, root cause, impact, lessons, and control improvements.

BCDR: Business Continuity & Disaster Recovery

Business continuity is proactive: how operations continue during disruption. Disaster recovery is reactive: how IT systems and data are restored after an incident.

BCP

Business Continuity Plan. Keeps critical business functions running before, during, and immediately after disruption.

DRP

Disaster Recovery Plan. Restores IT systems, applications, infrastructure, and data after an incident.

RTO

Recovery Time Objective. Maximum acceptable time to restore service.

RPO

Recovery Point Objective. Maximum acceptable amount of data loss measured in time.

Cyber Risk Management

Risk management identifies, prioritizes, manages, and monitors risks to information systems. Good exam answer: frame the context, assess threats and vulnerabilities, respond using acceptance/mitigation/remediation/transfer, then monitor continuously.

Risk formula
risk = likelihood x impact
mitigation = reduce likelihood or reduce impact
remediation = remove the weakness
transfer = shift financial impact, often with insurance
acceptance = tolerate low risk knowingly
Episode 06 / Privacy, Law & Cyber Warfare

Consent Forms, Nation States, And Other Plot Twists

Data privacy is about a person controlling how their personal data is collected, stored, used, shared, corrected, or deleted. Data security protects that data from unauthorized access and misuse. They are related, but not identical.

Data Privacy Principles

Access

People should know what personal data is held and be able to access or correct it.

Transparency

Organizations should explain what they collect, why, how it is used, and who it is shared with.

Consent

Personal data should be collected or processed with consent unless a lawful exception applies.

Data Minimization

Collect only what is needed, use it only for the stated purpose, and keep it only as long as necessary.

Privacy By Design

Privacy should be built into systems by default, not patched on later.

Security

Use IAM, MFA, encryption, DLP, backups, audit trails, and vendor controls to protect personal data.

India DPDP Act

The Digital Personal Data Protection Act, passed in August 2023, balances individual rights over digital personal data with lawful processing needs. It places obligations on Data Fiduciaries, gives rights and duties to Data Principals, and creates penalties for breaches.

Redacted Legal File

Remember the actors: Data Principal is the person the data belongs to. Data Fiduciary decides purpose and means of processing. classified filler The Data Protection Board oversees compliance and complaints.

Cyber Law In India: Exam Anchors

  • IT Act, 2000: India's foundational cyber law for electronic records, digital signatures, cyber offences, intermediary duties, and penalties.
  • IT Rules / SPDI Rules: earlier privacy and reasonable security practice rules for sensitive personal data.
  • DPDP Act, 2023: India's dedicated digital personal data protection framework.
  • Reporting: cybercrime complaints can be reported via the National Cyber Crime Reporting Portal and helpline 1930 for cyber financial fraud.

Cyber Warfare

Cyber warfare usually refers to cyberattacks by or for nation-state interests that target another country. It can involve espionage, sabotage, DoS, attacks on critical infrastructure, propaganda, economic disruption, or surprise attacks.

Espionage

Stealing secrets from government, military, industry, or research systems.

Sabotage

Destroying, altering, or disabling important information or infrastructure.

DoS / DDoS

Flooding critical services to make them unavailable to legitimate users.

Propaganda

Using digital channels to influence opinion, spread falsehoods, or damage trust.

Confidentiality vs Integrity?

Authentication verifies identity. Authorization grants permissions.

  • Authentication is not authorization.

  • BCP is not the same as DRP.
